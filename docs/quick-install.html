LewisWhisper — Quick Install

local voice dictation · new-Mac setup in five steps · Apple Silicon, macOS 14+

1

Download

Grab the latest release zip on the new Mac:

github.com/lewisitconsulting1/whisp/releases/latest → LewisWhisper-0.7.0.zip

2

Run the installer

Unzip, then right-click setup.command → Open → Open.

It installs the app, opens the Ollama download if needed, and pulls the cleanup model (gemma3:4b, ~3.3 GB, one time). Prefer manual? Drag LewisWhisper.app to Applications — it's notarized, it just opens.

3

Grant three permissions

The ⚠️ menu-bar icon walks you through: Microphone (click Allow) · Input Monitoring (toggle ON) · Accessibility (toggle ON).

Then quit the app (menu → Quit) and open it once more from Applications.

4

Let it load

First launch downloads the speech models (~2.5 GB) — the icon shows … for a few minutes, then becomes the soundwave. That means ready.

5

Dictate

Hold RIGHT OPTION (⌥), speak, release — clean text lands wherever your cursor is.

Quick-tap instead for hands-free; it stops when you stop talking. Icons: soundwave ready · 🔴 recording · ⏳ transcribing · ⚠️ needs permissions

Shortcut for a quick test — skip the Ollama download

New in 0.7.0: Settings (⌘,) → Cleanup AI can use a server you already have instead of installing models on this Mac. Pick Remote Ollama server and point it at your M4 Pro (http://<m4-name>.local:11434 — start Ollama there with OLLAMA_HOST=0.0.0.0 ollama serve), or pick a cloud provider and paste an API key (stored in the Keychain). Hit Test — a green ✓ means you're set. The speech models in step 4 are always local and still required.

If something's off

Uncleaned "um"-filled pastes → cleanup backend unreachable — open Settings and hit Test. Stuck ⚠️ after granting → quit & reopen; still stuck → remove the entry with − in System Settings and re-grant. Nothing on hotkey → make sure it's the right-side Option key.
